# v2.0.0: The Deep Context Engine

> **Your AI agent now understands your entire codebase — in four languages, at 3% of the token cost.**

---

## Why Upgrade?

Every `afd_read` call on a large file used to burn thousands of tokens. Now it returns a 97%-compressed structural skeleton — and if you read it again, Anthropic's prompt caching kicks in, so you pay near-zero on the second hit.

The immune system that started with TypeScript now speaks Python, Go, and Rust. Cross-file dependencies are traced to 3 levels deep. And you can see all of it happening live in a premium web dashboard that ships as a single HTML file.

**tl;dr** — afd 2.0 saves more tokens, understands more languages, and shows you exactly how much it's saving in real time.

---

## Highlights

### Deep Context Engine
4-language AST parsing via Tree-sitter WASM. Cross-file symbol tracing to L2 (direct imports) and L3 (transitive dependencies). Barrel file resolution for `index.ts` re-exports. JSX/TSX components detected in call graphs.

### True Caching
New `afd://hologram/{path}` MCP resource with `cache_control: { type: "ephemeral" }`. When Claude reads a hologram via `resources/read`, Anthropic's server caches it — subsequent reads of the same file cost near-zero tokens.

### Premium Web Dashboard
`afd web` opens a glassmorphism-styled dashboard in your browser. Overview tab shows today's savings, lifetime ROI, 7-day history, and live SSE events. Context Compressor tab lets you browse files, view syntax-highlighted holograms, and explore N-Depth dependency trees. Single HTML file, no CDN, no build step.

### Zero-Friction DX
- `afd web` — one command to open the dashboard
- Fixed port `51831` — predictable `localhost:51831/dashboard`
- Daemon watchdog — `daemonRequest()` retries 3x on transient failures
- `afd setup` — interactive 4-step project configuration

### Honest Metrics
The old `chars ÷ 4` token estimation is gone. Replaced by a content-aware heuristic engine with per-extension ratios (TypeScript 3.2, JSON 2.8, Markdown 4.2, etc.). Conservative by design — we'd rather undercount savings than overcount.
