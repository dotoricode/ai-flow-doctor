### Cross-daemon HTTP bridge вАФ Design

- Each daemon registers `{ workspace, port, pid }` into `~/.afd/mesh.json` on start
- Stale entries (dead PIDs) auto-pruned on every read/write
- New HTTP endpoint: `GET /mesh/peers` вАФ returns live peer list excluding self
- New client helper: `getMeshPeers()` in `src/daemon/client.ts`
- On daemon stop, entry removed from registry via `deregisterMesh()`
